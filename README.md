# Vyezdnik

**Въездник** — цифровой гид по правилам въезда в разные страны мира. Проект помогает пользователю быстро разобраться в визовых требованиях, обязательных анкетах, документах и ограничениях на въезд без необходимости вручную искать информацию на десятках официальных сайтов. Система объединяет **мультиагентный backend**, **RAG-поиск по базе знаний** и **чатовый интерфейс**, чтобы отвечать на вопросы о поездках в удобной, понятной и структурированной форме.

## Что делает проект

- отвечает на вопросы о правилах въезда в разные страны
- подсказывает, какие документы и формы могут понадобиться
- различает запросы про визовые правила и запросы про анкеты и документы
- собирает, парсит и подготавливает данные из внешних источников для базы знаний
- поддерживает многошаговый диалог с сохранением контекста
- собирает аналитику и технические метрики для мониторинга

## Основные возможности

### Подготовка и загрузка данных

Проект включает отдельный пайплайн подготовки данных:

- парсинг внешних источников с правилами въезда
- структурирование и очистка данных
- формирование базы знаний по странам
- загрузка подготовленных данных в Qdrant для последующего RAG-поиска

### Мультиагентная архитектура

В backend используются несколько специализированных агентов:

- `guard agent` — проверяет, относится ли запрос к теме поездок и въезда
- `router agent` — определяет тип запроса
- `lawyer agent` — отвечает по правилам въезда, визам и ограничениям
- `form agent` — помогает с формами, анкетами и документами
- `orchestrator` — объединяет ответы агентов в итоговый ответ пользователю

### Retrieval-Augmented Generation

Проект использует RAG-подход для поиска информации:

- база знаний по странам хранится в Markdown
- метаданные и формы хранятся в JSON
- поиск по базе реализован через Qdrant
- для поиска релевантных фрагментов используются эмбеддинги

### Fullstack-решение

Проект включает в себя:

- `frontend` на Next.js
- `backend` на FastAPI
- `redis` для хранения истории диалога
- `monitoring` на Prometheus и Grafana
- `docker-compose` для локального запуска всех сервисов

## Локальный запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/ya-vera/vyezdnik2.0.git
cd vyezdnik2.0
```

### 2. Подготовка базы знаний

Сначала поднимается Qdrant, после чего выполняются скрипты парсинга и загрузки данных:

```bash
sudo docker compose up -d qdrant
python backend/scripts/parsing.py
python backend/scripts/ingest.py
```

### 3. Запуск приложения

После подготовки базы можно поднять остальные сервисы проекта:

```bash
sudo docker compose up --build -d
```

## Как работает система

Пользователь отправляет вопрос через интерфейс приложения, после чего backend определяет тип запроса и маршрутизирует его между специализированными агентами. Для ответа система использует заранее подготовленную базу знаний по странам: данные собираются и обрабатываются отдельными скриптами парсинга и ingestion, после чего загружаются в векторную базу. Затем backend извлекает релевантную информацию, формирует итоговый ответ и возвращает его пользователю в чат. История диалога сохраняется для поддержки многошагового взаимодействия, а технические метрики собираются для мониторинга и анализа работы сервиса.

## Источники данных

В основе проекта лежит база знаний по странам, собранная из открытых и структурированных источников. Для каждой страны используются текстовые материалы с правилами въезда, а также метаданные и ссылки на формы и документы, необходимые пользователю перед поездкой. Сбор, парсинг, очистка и загрузка этих данных выполняются отдельными скриптами подготовки данных, после чего информация индексируется в Qdrant для последующего поиска.

## Ограничения

Проект предоставляет справочную информацию и не заменяет официальные государственные и консульские источники. Перед поездкой пользователю рекомендуется дополнительно проверять актуальные требования на сайтах посольств, миграционных служб, авиакомпаний и официальных государственных порталов.

## Почему этот проект полезен

Информация о въезде в страну часто разрознена, быстро меняется и находится в неудобных для пользователя источниках. **Въездник** помогает сократить время на поиск и сделать ответы более доступными, используя LLM-подход, поиск по базе знаний и мультиагентную маршрутизацию.
